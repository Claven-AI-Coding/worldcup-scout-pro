## 💡 节省成本的方法

### 1. 优化工作流执行时间

**当前配置：** ~12.5 分钟

**优化方案：**
```yaml
# 并行执行测试
jobs:
  frontend-test:
    runs-on: ubuntu-latest
    # ...
  
  backend-test:
    runs-on: ubuntu-latest
    # ...
  
  # 这两个任务会并行执行，而不是顺序执行
```

**优化后：** ~8-10 分钟（节省 20-30%）

### 2. 使用缓存

```yaml
- name: Cache dependencies
  uses: actions/cache@v3
  with:
    path: ~/.npm
    key: ${{ runner.os }}-npm-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-npm-
```

**效果：** 节省 1-2 分钟

### 3. 只在必要时部署

```yaml
on:
  push:
    branches: [main]
    paths:
      - 'backend/**'
      - 'frontend/**'
      - '.github/workflows/cd.yml'
  # 不会因为文档更新而触发部署
```

### 4. 使用更快的运行器

```yaml
runs-on: ubuntu-latest      # 标准（免费）
runs-on: ubuntu-latest-4x   # 4 核（付费）
```

**建议：** 使用免费的标准运行器

## 🔐 成本控制最佳实践

### 1. 设置支出限制

```
Settings → Billing and plans → Spending limit
设置为 $10 或更低
```

### 2. 监控使用情况

- 每周检查一次使用情况
- 如果接近限制，优化工作流

### 3. 使用工作流条件

```yaml
# 只在特定条件下运行
if: github.event_name == 'push' && github.ref == 'refs/heads/main'
```

### 4. 定期审查

- 每月审查一次工作流
- 删除不必要的工作流
- 优化长时间运行的任务

## 📞 常见问题

**Q: 如果超出免费额度会怎样？**  
A: GitHub 会自动停止工作流，除非你设置了支出限制。

**Q: 如何避免超出额度？**  
A: 设置支出限制，定期监控使用情况，优化工作流。

**Q: 自托管运行器需要付费吗？**  
A: 不需要，但需要自己维护服务器。

**Q: 可以取消自动部署吗？**  
A: 可以，删除工作流文件或禁用工作流即可。
