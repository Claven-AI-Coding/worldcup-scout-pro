## 📝 知识库维护的规范

### 知识提交规范

```
格式：
- 标题：清晰简洁
- 内容：结构化、易理解
- 标签：准确分类
- 日期：记录时间
- 作者：记录贡献者
- 版本：跟踪更新

示例：
标题：Claude Opus 模型在代码审查中的应用
标签：#模型使用 #代码审查 #最佳实践
日期：2026-03-14
作者：CodeReviewer
版本：1.0
```

### 知识审核规范

```
审核标准：
1. 准确性：信息是否正确
2. 完整性：是否包含必要信息
3. 清晰性：是否易于理解
4. 相关性：是否与知识库相关
5. 时效性：是否仍然有效

审核流程：
1. 初审（KnowledgeKeeper）
2. 复审（Claven）
3. 最终审批（GraySon，如需要）
```
