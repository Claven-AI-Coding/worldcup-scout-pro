## 🧠 模型选择策略

### 模型分类

**Opus Thinking（最强，最贵）**
- 用途：复杂决策、架构设计、安全分析
- 成本：最高
- 速度：最慢
- 适合：关键决策、一次性分析

**Opus 4.5（强，贵）**
- 用途：复杂代码、深度分析
- 成本：高
- 速度：中等
- 适合：核心功能开发

**Sonnet 4.5（平衡，中等）**
- 用途：一般开发、代码审查
- 成本：中等
- 速度：快
- 适合：日常开发工作

**Haiku（快，便宜）**
- 用途：简单任务、文档、协调
- 成本：最低
- 速度：最快
- 适合：高频简单任务

### 成本优化公式

```
总成本 = (Opus-Thinking × 5%) + (Opus × 15%) + (Sonnet × 40%) + (Haiku × 40%)

目标：在保证质量的前提下，最小化成本
```

## 📋 多项目管理框架

### 项目类型

**同步项目（Synchronous）**
- 特点：需要实时协调、依赖关系强
- 例如：世界杯球探 Pro（当前项目）
- 工作模式：紧密协作、日报告

**异步项目（Asynchronous）**
- 特点：相对独立、可以并行进行
- 例如：其他产品开发
- 工作模式：周报告、里程碑制

### 项目配置示例

```yaml
projects:
  - name: "worldcup-scout-pro"
    type: "synchronous"
    priority: "P0"
    agents:
      - Claven (Leader)
      - CodeSmith
      - UIArtisan
      - TestMaster
      - DevOpsGuru
    status: "active"
    deadline: "2026-03-31"
    
  - name: "product-b"
    type: "asynchronous"
    priority: "P1"
    agents:
      - CodeSmith (part-time)
      - UIArtisan (part-time)
    status: "planning"
    deadline: "2026-04-30"
    
  - name: "product-c"
    type: "asynchronous"
    priority: "P2"
    agents:
      - RequirementsAnalyst
      - ProjectManager
    status: "planning"
    deadline: "2026-05-31"
```

```
你是 AI Agent 矩阵的 Leader。你需要为每个任务分配最优的模型。

3. 速度考虑：
   - 同步项目：优先速度（Sonnet/Haiku）
   - 异步项目：可以用 Opus（质量优先）
   - 关键路径：优先质量（Opus）

4. 任务特性：
   - 代码生成：Sonnet 4.5
   - 代码审查：Sonnet 4.5
   - 架构设计：Opus Thinking
   - 安全分析：Opus 4.5
   - 文档生成：Haiku
   - 简单协调：Haiku
   - 数据分析：Sonnet 4.5
   - 性能优化：Sonnet 4.5

分配时考虑：
- 任务复杂度
- 项目优先级
- 当日成本预算
- Agent 当前负载
- 交付时间要求
```
