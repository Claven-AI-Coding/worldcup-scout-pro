## ❓ 问题 1：Claven 是否可以自行切换到新的对话？

## 📋 Claven 自主切换的规则

### 何时应该切换

**1. 项目切换时**
```
当前项目：世界杯球探 Pro
新项目：产品 B 开发

触发条件：
  - 收到新项目的需求
  - 当前项目进入维护阶段
  - 需要并行处理多个项目

行动：
  /new
  记录当前项目的状态
  开始新项目的对话
```

**2. 任务类型完全不同时**
```
当前任务：后端 API 开发
新任务：前端界面设计

触发条件：
  - 任务类型完全不同
  - 需要不同的 Agent 团队
  - 需要不同的模型配置

行动：
  /reset
  切换 Agent 分配
  调整模型配置
```

**3. 上下文过于复杂时**
```
当前上下文：100+ 条对话
新需求：简单的文档生成

触发条件：
  - 对话历史超过 100 条
  - Token 使用接近上限
  - 需要清晰的新开始

行动：
  /compact "保留关键决策"
  或 /new
  重新开始
```

**4. 定期维护时**
```
每周五下午：
  - 总结本周工作
  - 压缩会话历史
  - 准备下周计划

行动：
  /status
  /compact
  /stop
  准备下周的 /new
```

## 📊 自主切换的标准流程

### 流程图

```
任务完成或新需求到达
    ↓
Claven 评估情况
    ↓
判断是否需要切换
    ├─ 是 → 检查切换类型
    │       ├─ 项目切换 → /new
    │       ├─ 任务切换 → /reset
    │       ├─ 历史压缩 → /compact
    │       └─ 会话结束 → /stop
    │
    └─ 否 → 继续当前会话
```

### 具体步骤

**步骤 1：评估当前状态**
```
检查项目：
  - 当前项目是什么？
  - 当前任务是什么？
  - 进度如何？

检查资源：
  - Token 使用情况
  - 成本情况
  - 时间情况
```

**步骤 2：判断是否需要切换**
```
问题 1：是否有新项目？
  是 → 项目切换（/new）
  否 → 继续

问题 2：是否有新任务？
  是 → 任务切换（/reset）
  否 → 继续

问题 3：历史是否过长？
  是 → 压缩历史（/compact）
  否 → 继续

问题 4：是否完成工作？
  是 → 停止会话（/stop）
  否 → 继续
```

**步骤 3：执行切换**
```
如果需要切换：
  1. 记录当前状态
  2. 保存重要信息
  3. 执行切换命令
  4. 通知 GraySon（如需要）
  5. 开始新会话
```

**步骤 4：通知 GraySon**
```
切换时的通知格式：

🔄 会话切换通知

从：[当前项目/任务]
到：[新项目/任务]

原因：[切换原因]

当前状态：
  - Token 使用：XX%
  - 成本：$XX
  - 完成度：XX%

新会话配置：
  - 模型：[选择的模型]
  - 思考深度：[设置]
  - 详细模式：[on/off]

预计时间：[预计完成时间]
```

## 📝 任务分配的日常流程

### 每天早上 09:00

```
1. 检查待处理任务
   /status
   查看 token 和成本

2. 分析优先级
   - P0 任务优先
   - 有依赖关系的任务
   - 时间紧张的任务

3. 分配任务给 Agent
   - CodeSmith：后端开发
   - UIArtisan：前端开发
   - TestMaster：测试
   - 其他 Agent：按需

4. 发送任务分配单
   - 明确的目标
   - 清晰的要求
   - 具体的标准

5. 确认 Agent 理解
   - 等待 Agent 确认
   - 回答 Agent 的问题
   - 澄清不清楚的地方
```

### 每天中午 12:00

```
1. 检查进度
   - 各 Agent 的完成情况
   - 是否有卡点
   - 是否需要调整

2. 解决问题
   - 帮助 Agent 解决卡点
   - 调整任务优先级
   - 重新分配资源

3. 更新计划
   - 根据进度更新计划
   - 调整下午的任务
   - 准备明天的任务
```

### 每天下午 17:00

```
1. 汇总成果
   - 收集各 Agent 的成果
   - 验收完成的任务
   - 记录问题和改进

2. 生成日报告
   - 今天完成了什么
   - 遇到了什么问题
   - 明天的计划

3. 准备明天
   - 分析明天的任务
   - 准备任务分配单
   - 预估资源需求
```
