## 🔍 问题描述

### 现象

当在输入框中输入 `/` 时，系统应该显示可用的命令提醒。但是 `/strategy` 命令没有出现在提醒列表中。

### 原因分析

`/strategy` 命令是我们为了支持动态会话策略切换而设计的新命令，但它还没有被集成到 OpenClaw 系统中。

### 影响范围

- ❌ 无法通过 `/strategy` 命令动态切换会话策略
- ❌ 无法在输入时看到 `/strategy` 的提醒
- ❌ 无法使用暂停-恢复、完全隔离、并行活跃三种模式
- ❌ 影响多项目管理的灵活性

---

## 📋 需要实现的命令

### 策略管理命令

```
/strategy pause-resume
  - 切换到暂停-恢复模式
  - 当前会话进入暂停状态
  - 不占用 token
  - 可以随时恢复

/strategy complete-isolation
  - 切换到完全隔离模式
  - 当前会话完全结束
  - 需要重新加载上下文
  - 适合异步项目

/strategy parallel-active
  - 切换到并行活跃模式
  - 多个会话同时活跃
  - 占用更多 token
  - 适合高频切换

/strategy status
  - 查看当前策略
  - 查看所有活跃会话
  - 查看会话状态

/strategy list
  - 列出所有可用策略
  - 显示每个策略的特点
```

### 会话管理命令

```
/switch <project-name>
  - 切换到指定项目
  - 仅在并行活跃模式下有效

/switch-session <project-name> <session-id>
  - 切换到指定项目的指定会话
  - 仅在完全隔离模式下有效

/pause [project-name]
  - 暂停当前或指定会话
  - 仅在暂停-恢复模式下有效

/resume <project-name>
  - 恢复暂停的会话
  - 仅在暂停-恢复模式下有效

/list-sessions
  - 列出所有会话
  - 显示会话状态
  - 显示会话信息

/status-all
  - 查看所有会话状态
  - 显示每个会话的进度和 token 使用

/stop-all
  - 结束所有会话
  - 所有会话都保存
```
