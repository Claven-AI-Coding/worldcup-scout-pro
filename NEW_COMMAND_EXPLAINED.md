## 🔄 /new 命令的行为

### 什么会被清空

```
/new 会清空：
✅ 当前会话的对话历史
✅ 当前会话的上下文
✅ 当前会话的临时变量
✅ 当前会话的工作状态

/new 不会清空：
❌ 文件系统中的文件
❌ Git 仓库中的代码
❌ 内存文件（memory/*.md）
❌ 知识库中的文档
❌ 已提交的代码
```

### 具体例子

```
会话 A：
  - 对话 1：讨论架构
  - 对话 2：编写代码
  - 对话 3：修复 Bug
  
执行 /new
  ↓
会话 B：
  - 完全新的会话
  - 看不到会话 A 的历史
  - 但会话 A 的所有工作成果仍然存在
```

## 🎯 /new 的实际含义

### 不是"删除"

```
❌ /new 不是删除所有数据
❌ /new 不是清空硬盘
❌ /new 不是丢失工作成果
```

### 而是"重新开始"

```
✅ /new 是清空当前会话的对话历史
✅ /new 是重置当前会话的上下文
✅ /new 是开始一个新的对话线程
✅ 但所有工作成果都被保存了
```

### 类比

```
/new 就像：
- 关闭一个浏览器标签页
- 但你的文件仍然在硬盘上
- 你可以打开新标签页重新访问

/new 不像：
- 删除文件
- 格式化硬盘
- 清空数据库
```
